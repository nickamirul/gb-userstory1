# Math Calculator Backend

A robust Node.js/Express HTTPS backend server for performing mathematical calculations with API key authentication and SSL/TLS encryption.

## Project Description

This backend server provides a secure HTTPS REST API for mathematical calculations with proper operator precedence handling. It supports basic arithmetic operations (+, -, *, /) with multiplication and division having higher precedence than addition and subtraction. The server includes SSL/TLS encryption, comprehensive input validation, error handling, and API key authentication for secure access.

## API Endpoints

### Calculate Expression
```
POST https://localhost:3001/api/v1/calculate
Headers: X-API-Key: math-calculator-api-key
Content-Type: application/json

Body:
{
  "expression": "2 + 3 * 4"
}
```

### Health Check
```
GET https://localhost:3001/api/v1/health
```

## Authentication

All API endpoints require an API key in the request headers:

```
X-API-Key: math-calculator-api-key
```

Or alternatively:
```
Authorization: math-calculator-api-key
```

## ðŸ”’ HTTPS Setup

This backend requires SSL certificates for HTTPS operation.

### Prerequisites

1. **Install mkcert** (if not already installed):
   ```bash
   # macOS
   brew install mkcert
   mkcert -install
   
   # Generate certificates in project root
   mkdir -p ../certs
   mkcert -key-file ../certs/localhost-key.pem -cert-file ../certs/localhost.pem localhost 127.0.0.1 ::1
   ```

2. **Create `.env` file**:
   ```bash
   # Create .env file in backend directory
   cat > .env << EOF
   PORT=3001
   NODE_ENV=development
   API_KEY=math-calculator-api-key
   EOF
   ```

### SSL Certificate Files

The server expects these SSL certificate files:
- `../certs/localhost.pem` - SSL certificate
- `../certs/localhost-key.pem` - Private key

These files are automatically generated by mkcert and are git-ignored for security.

6. **Run Tests**:

The server will start on `https://localhost:3001` with SSL encryption.

## API Response Format

### Success Response
```json
{
  "success": true,
  "expression": "2 + 3 * 4",
  "result": 14,
  "timestamp": "2024-01-01T12:00:00.000Z"
}
```

### Error Response
```json
{
  "success": false,
  "error": "Division by zero is not allowed",
  "code": "DIVISION_BY_ZERO",
  "expression": "5 / 0"
}
```

## Error Codes

- `MISSING_API_KEY`: API key not provided
- `INVALID_API_KEY`: Invalid API key
- `MISSING_EXPRESSION`: Math expression not provided
- `INVALID_EXPRESSION_TYPE`: Expression is not a string
- `EMPTY_EXPRESSION`: Expression is empty
- `EXPRESSION_TOO_LONG`: Expression exceeds maximum length
- `DIVISION_BY_ZERO`: Attempt to divide by zero
- `INVALID_CHARACTER`: Expression contains invalid characters
- `MISMATCHED_PARENTHESES`: Parentheses are not properly matched
- `INVALID_EXPRESSION_FORMAT`: Expression format is invalid
- `CALCULATION_ERROR`: Generic calculation error
- `NOT_FOUND`: Endpoint not found
- `INTERNAL_ERROR`: Internal server error

## Configuration

### Environment Variables

The server uses a `.env` file for configuration:

```env
# Server Configuration
PORT=3001                        # HTTPS server port
NODE_ENV=development             # Environment mode

# Security
API_KEY=math-calculator-api-key  # API authentication key
```

### Configuration Priority

1. **Environment variables** (`.env` file) - **Highest Priority**
2. **Fallback configuration** (`https.config.js`) - Used if `.env` is missing

### SSL Certificate Configuration

The server automatically loads SSL certificates from:
- **Certificate**: `../certs/localhost.pem`
- **Private Key**: `../certs/localhost-key.pem`

These paths are configured in `server.js` and are relative to the backend directory.
